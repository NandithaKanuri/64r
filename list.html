1.What is HTML?

    • HTML stands for HyperText Markup Language. It’s the standard language used to create and structure content on the web.
    • Think of it as the skeleton of a webpage — it tells the browser what is on the page (headings, paragraphs, images, links, etc.), but not necessarily how it looks (that’s where CSS comes in)."
    • Here’s the gist:
      •
    • HyperText→Text that links to other text or resources (like hyperlinks).
    • Markup→ Special tags that describe the structure of the content.
    • Language→A set of rules for writing those tags.

    2.What are the difference between XHTML and HTML?

    • XHTML
      •

      XHTML stands for eXtensible HyperText Markup Language.It’s basically HTML rewritten to follow XML rules — meaning it’s stricter, more structured, and less forgiving about mistakes.

    • Rules:Very strict (follows XML rules).
    • Tag Closing:All tags must be closed.
    • Case Sensitivity:Tags & attributes must be lowercase
    • Doctype:XML-based Doctype
    • HTML
      •

      HTML is the standard markup language used to create web pages.HTML is lenient:browsers are forgiving if you make small mistakes like missing closing tags or not using lowercase letters.

    • Rules:More relaxed
    • Tag Closing: Optional for some tags
    • Case Sensitivity:Not case sensitive
    • Doctype:!DOCTYPE html(HTML5)

    3.What is a tag?

      In HTML (and XHTML), a tag is a special keyword enclosed in angle brackets (< >) that tells the browser how to display content. Tags usually come in pairs (start tag + end tag) or sometimes stand alone.The start tag tells the browser "start applying this rule," and the end tag tells it "stop here.

    • Paired Tags
    • Have both an opening tag and a closing tag.
    • Enclose some content between them.
    • Closing tag has a forward slash (/) before the name.
    • Unpaired Tags
    • Do not wrap any content.
    • They stand alone and don’t need a closing tag.
    • In HTML, they can be written without the /.In XHTML, they must be written as self-closing(/tagname>)

    4.What are elements?

      In HTML (and XHTML), an element is the complete building block of a webpage.It’s not just the tag — it’s the tag + content + closing tag (or just the tag if it’s empty).There are 3 types of elements:

    • Block elements:
    • Occupy the full width available.
    • Always start on a new line.
    • You can set width, height, margins, and padding.
    • Inline elements
    • Only take up as much width as the content inside them.
    • Stay on the same line with other inline elements.
    • You cannot set width/height directly.
    • Inline-block elements
    • Behave like inline elements (can sit next to each other).
    • But you can set width, height, margins, and padding like block elements.

    5.What are the differences between HTML4 and HTML5?

    • HTML4
      •

      HTML4 (HyperText Markup Language, Version 4) is a markup language standard released by the W3C in December 1997 and revised in 1999.It was the most widely used version of HTML before HTML5 came along.

    • Doctype Declaration:Long and specific to the DTD (Document Type Definition).
    • Structure & Semantics:Relied on div and span for layout.
    • Syntax Rules:Stricter attribute rules, required type for scripts/styles.
    • APIs:Mainly DOM scripting.DOM scripting in HTML4 refers to using JavaScript to interact with the Document Object Model (DOM) of a webpage — meaning you can read, change, add, or remove elements and their content dynamically after the page loads.
    • HTML5
      •

      HTML5 (HyperText Markup Language, Version 5) is the latest standard of HTML, officially released by the W3C in October 2014. It was designed to make the web more powerful, semantic, multimedia-rich, and mobile-friendly without needing extra plugins like Flash.

    • Doctype Declaration:Short and easy to remember
    • Structure & Semantics:Introduced semantic elements like header,main,footer nav etc.
    • Syntax Rules:Attributes like checked or disabled don’t need values.Quotes optional in some cases.Tags are case-insensitive
    • APIs:Added many built-in APIs like Geolocation API,Web Storage (localStorage, sessionStorage),Web Workers,Drag-and-Drop API,Offline capabilities (Service Workers)

    6.What are lists?

      In HTML, lists are used to organize related items so they’re easier to read, navigate, and understand.Here are three main types of lists in HTML:

    • Ordered Lists:
    • Items appear in a specific order, usually numbered.Used for step-by-step instructions, rankings, or sequences.
    • Unordered Lists:
    • Items have no particular order; shown with bullets by default.Used for collections, categories, or things without ranking.
    • Description Lists:
    • These lists are used to define terms and their corresponding descriptions.Displays terms and their descriptions (like a glossary).

    • HTML(HyperText Markup Language)
      •

      A markup language used to create and structure webpages and present content in web browsers.It uses predefined tags to define the layout, text formatting, images, links, and other elements for display.

    • Purpose:Designed to display data in web browsers.
    • Focus:How the data looks (presentation + structure).
    • Predefined tags : Comes with a fixed set of tags like
    • Syntax:Flexible — browsers often fix mistakes automatically
    • XML(Extensible Markup Language)
      •

      A markup language designed to store, transport, and organize data in a structured, human-readable, and machine-readable format. It uses custom tags defined by the user, and follows strict syntax rules to ensure the data is well-formed.

    • Purpose:Purpose: Designed to store and transport data, not display it.
    • Focus: What the data is (meaning/content).
    • Custom tags:You create your own tags to describe your data
    • Syntax:Strict — must follow well-formed rules (proper nesting, lowercase, quoted attributes, closed tags).

    8.What are attributes?

    In HTML, attributes are like extra instructions you attach to a tag to control its behavior, give it additional information, or change how it appears.

    some of the attribute sare listed below:

    • href:The href attribute in HTML is used to specify the destination URL (or file path) for a link.
    • targetThe target attribute in HTML specifies where the linked document will open when a user clicks on a link (or submits a form).
    • width and height:In HTML, the width and height attributes specify the display size of certain elements, most commonly images.
    • download:In HTML, the download attribute is used in an anchor tag to tell the browser to download the linked file instead of opening it.
    • id:In HTML, the id attribute is used to give an element a unique identifier on a webpage..
    • src:In HTML, the src attribute (short for “source”) tells the browser where to find the file that an element should load.
    • alt:In HTML, the alt attribute (short for "alternative text") provides a text description for an image when it can’t be displayed.
    • title:In HTML, the title attribute is used to provide extra information about an element.
    • type:In HTML, the type attribute specifies the type or format of the element’s content or behavior.
